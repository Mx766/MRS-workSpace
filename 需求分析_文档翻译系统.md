# 外文说明书翻译工具 — 需求分析

> 版本: v2.0 | 日期: 2026-06-23 | 状态: 待评审

---

## 一、项目定位

一个**简易封装程序**，面向公司内部员工，将外文说明书（英文为主）翻译为纯中文，保留原版格式。

核心原则：
- **够用不复杂** — 单机程序，不搞分布式
- **翻译可干预** — 术语库控制 + 对话式审校
- **格式不走样** — PDF 和 Word 都保持原版排版

---

## 二、功能需求

### 2.1 API 密钥配置

程序启动时提供 **API 设置界面**：

```
┌─────────────────────────────────────┐
│  API 设置                            │
│                                     │
│  服务商: [DeepSeek ▼]               │
│  API Key: [________________________]│
│  模型:    [deepseek-chat ▼]         │
│                                     │
│  [测试连接]          [保存]          │
└─────────────────────────────────────┘
```

- 支持多家服务商切换（DeepSeek / OpenAI / 本地 Ollama）
- Key 加密存储到本地配置文件，不硬编码
- 提供连接测试按钮

### 2.2 术语库管理（万条级）

**存储**：SQLite 单文件数据库，支持万条以上术语。

**Excel 批量导入**：
```
┌──────────────────────────────────────────────┐
│  术语库管理                                    │
│                                              │
│  [下载模板]  [导入Excel]  [导出CSV]  [+添加]   │
│                                              │
│  ┌──────────────────────────────────────────┐ │
│  │ 搜索: [____________]  领域: [全部 ▼]      │ │
│  ├────┬──────────────┬──────────┬───────────┤ │
│  │ #  │ 英文术语      │ 中文翻译   │ 领域      │ │
│  ├────┼──────────────┼──────────┼───────────┤ │
│  │ 1  │ cyclic olefin│ 环烯烃共聚物│ 化学     │ │
│  │ 2  │ neocollagen..│ 新胶原生成 │ 医学     │ │
│  │ .. │ ...          │ ...       │ ...       │ │
│  └────┴──────────────┴──────────┴───────────┘ │
│  共 8,234 条                   第 1/412 页    │
└──────────────────────────────────────────────┘
```

- **导入**：Excel (.xlsx)，自动去重、冲突提示
- **添加**：单条新增，即时生效
- **导出**：CSV 格式，兼容 PDFMathTranslate
- **匹配**：精确匹配 + 词形还原（复数/时态归一化）
- **保留规则**：支持正则（如 `CAS \d+-\d+-\d+` → 保留不译）

### 2.3 文档翻译

#### 输入
- PDF（数字版 + 扫描件）
- Word (.docx)

#### 输出
- **同时导出 PDF 和 Word 两份**，纯中文，不生成双语对照
- Word 版供人工编辑，PDF 版直接使用
- PDF 翻译通过 pdf2zh 生成中文 PDF，Word 翻译通过 python-docx 生成格式一致的 DOCX

#### 翻译流程

```
源文件 → 解析(布局检测/OCR) → 术语库预匹配 → API翻译 → 术语后处理 → 输出
                                                          ↓
                                              库中未匹配术语 → 标记输出
```

#### 图片型 PDF
- 检测到扫描件时自动切换 OCR 管线（MinerU/PaddleOCR）
- OCR 结果标记置信度，低置信度文本标红提醒人工复核

### 2.4 未匹配术语标记

翻译过程中，遇到术语库中**没有**的专业词汇：

1. **自动收集**：提取高频未知词汇
2. **在译文中标记**：用 `{?术语?}` 或高亮标注
3. **审校时提示**：「以下术语未在库中找到，请确认翻译：xxx, yyy...」
4. **一键入库**：确认翻译后直接添加到术语库

### 2.5 对话式审校

翻译完成后，进入**对话修改模式**：

```
审校 > 第3页的"消防员防护应包括自给式呼吸器"改成"消防人员防护须包含自给式呼吸器"
  ✅ 已修改。

审校 > 全文的"colour"统一改成"颜色"
  ✅ 全文共 12 处已替换。

审校 > 检查第5页表格，单位 mm Hg 应该保留不翻译
  ✅ 第5页表格中 3 处 "毫米汞柱" 已还原为 "mm Hg"。

审校 > 把所有带 {?} 标记的术语处理一下
  📋 检测到 8 个未匹配术语：
     1. olefin copolymer → 当前: 烯烃共聚物 [确认/修改]
     2. carboxyhemoglobin → 当前: ___ [请输入翻译]
     ...
```

**支持的操作**：
- 指定页码/段落修改
- 全文查找替换
- 批量处理未匹配术语
- 修改后自动更新术语库

## 三、技术方案

### 3.1 整体架构

```
┌────────────────────────────────────────┐
│           Gradio 简易 Web UI            │
│   (配置页 / 翻译页 / 术语库 / 审校页)    │
├────────────────────────────────────────┤
│              核心引擎                    │
│  ┌──────────┬──────────┬────────────┐  │
│  │PDF翻译   │Word翻译  │ OCR(扫描件) │  │
│  │pdf2zh    │python-docx│MinerU      │  │
│  └──────────┴──────────┴────────────┘  │
│  ┌──────────┬──────────┐               │
│  │ 术语匹配  │ API调用   │               │
│  │ SQLite   │ DeepSeek  │               │
│  └──────────┴──────────┘               │
├────────────────────────────────────────┤
│              数据存储                    │
│  config.json   glossary.db   缓存/      │
└────────────────────────────────────────┘
```

### 3.2 技术选型

| 组件 | 方案 | 理由 |
|------|------|------|
| PDF 翻译内核 | PDFMathTranslate (pdf2zh) | 布局保留最好 |
| Word 翻译 | python-docx run 级替换 | 格式完全保留 |
| 翻译 API | DeepSeek / OpenAI 兼容 | 性价比高，支持术语注入 |
| 术语库 | SQLite | 单文件、万条级、免安装 |
| OCR | MinerU / PaddleOCR | 开源、中文支持好 |
| UI | Gradio | 纯 Python、开发快、够用 |

### 3.3 术语注入策略

```
1. 翻译前：匹配的术语 → 注入 system prompt
   "以下术语必须按指定翻译：olefin copolymer → 环烯烃共聚物"

2. 翻译后：正则扫描译文 → 强制替换（兜底）
   防止模型漏掉术语表中的规定翻译

3. 未匹配词检测：
   - 对原文做分词/NER
   - 过滤常见词（停用词表）
   - 剩下的低频专业词 → 标记为待确认
```

### 3.4 审校对话实现

```
用户输入 → LLM 解析意图(改哪/改什么/怎么改) → 
  ├── 定位修改：确定页码范围 → 重新渲染该页
  ├── 全文替换：find & replace → 批量更新
  └── 术语入库：确认翻译 → 写入 glossary.db
```

### 2.6 翻译历史与版本管理

每次翻译自动保存记录，支持回溯和清理：

- **自动保存**：翻译完成自动记录 — 原文 + 译文(PDF+Word) + 术语表快照 + 审校记录
- **随时回溯**：历史列表点击即可下载当时任一版本
- **按时间清理**：一键删除 N 天前的旧记录，释放空间
- **容量显示**：实时显示历史占用空间
